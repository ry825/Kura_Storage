# タスクリスト

## 🚨 タスク完全完了の原則

**このファイルの全タスクは最終的に完了させる。ただし、1回の実装では1つのPull Request単位を完了し、Pull Request作成後に停止してよい。**

### 必須ルール
- **全てのタスクを`[x]`にすること**
- 「時間の都合により別タスクとして実施予定」は禁止
- 「実装が複雑すぎるため後回し」は禁止
- 選択したPull Request単位に未完了タスク（`[ ]`）を残したまま作業を終了しない
- 後続のPull Request単位や後続フェーズのタスクは、`[ ]`のまま残してよい

### 実装可能なタスクのみを計画
- 計画段階で「実装可能なタスク」のみをリストアップ
- 「将来やるかもしれないタスク」は含めない
- 「検討中のタスク」は含めない

### タスクスキップが許可される唯一のケース
以下の技術的理由に該当する場合のみスキップ可能:
- 実装方針の変更により、機能自体が不要になった
- アーキテクチャ変更により、別の実装方法に置き換わった
- 依存関係の変更により、タスクが実行不可能になった

スキップ時は必ず理由を明記:
```markdown
- [x] ~~タスク名~~（実装方針変更により不要: 具体的な技術的理由）
```

### タスクが大きすぎる場合
- タスクを小さなサブタスクに分割
- 分割したサブタスクをこのファイルに追加
- サブタスクを1つずつ完了させる

### Pull Request運用
- フェーズは上から順番に実施し、同時に複数フェーズへ着手しない
- 実装開始前に、現在のフェーズの未完了項目から、1つの変更目的として実装・テスト・レビューできる範囲を1つのPull Request単位として選ぶ
- 強く依存する項目は同じPull Requestにまとめ、無関係な機能やRefactorは分ける
- tasklist.mdのチェック項目ごとに機械的にBranchやPull Requestを作成しない
- Pull Request単位ごとに、原則として最新の`main`から短命な作業Branchを作成する
- 選択範囲の実装、テスト、tasklist.md更新、Commit、Pushが完了したらPull Requestを作成する
- コーディングエージェントはPull RequestをMergeせず、Pull Request作成後に停止する
- 次のPull Request単位は、ユーザーから実施指示を受けた後に開始する
- 次の範囲が未MergeのPull Requestへ依存する場合は、依存元が`main`へMergeされた後に開始する

## フェーズ3: 品質確認と修正

> このフェーズには、対象プロジェクトと今回の変更に適用される確認項目のみを記載する。
> 使用しない言語、ツール、実行環境に関する項目は、計画作成時点で追加しない。
> `{確認コマンド}`や`{対象機能}`などのプレースホルダーは、実際の内容へ置き換えてから作業を開始する。

### 自動チェック

* [ ] コードフォーマットの確認が完了している

  * [ ] `{フォーマット確認コマンド}`が成功する
* [ ] 静的解析の確認が完了している

  * [ ] `{静的解析コマンド}`がエラーなく成功する
* [ ] ビルドまたはコンパイルが成功する

  * [ ] `{ビルドコマンド}`が成功する
* [ ] 自動テストが成功する

  * [ ] `{単体テストコマンド}`が成功する
  * [ ] `{結合テストコマンド}`が成功する
  * [ ] `{E2Eテストコマンド}`が成功する
* [ ] パッケージまたは成果物を生成する変更の場合、生成処理が成功する

  * [ ] `{パッケージングまたは成果物生成コマンド}`が成功する

### 機能確認

* [ ] 今回変更した主要な機能が要件どおり動作する

  * [ ] `{正常系の確認内容}`
  * [ ] `{入力値や状態の境界条件の確認内容}`
  * [ ] `{エラー発生時の確認内容}`
* [ ] 既存機能への影響がないことを確認する

  * [ ] `{関連する既存機能の確認内容}`
* [ ] ユーザーインターフェースを変更した場合、対象画面を手動確認する

  * [ ] `{画面表示の確認内容}`
  * [ ] `{主要操作の確認内容}`
  * [ ] `{エラー表示の確認内容}`
* [ ] APIを変更した場合、対象APIを確認する

  * [ ] `{正常なリクエストとレスポンスの確認内容}`
  * [ ] `{不正なリクエストの確認内容}`
  * [ ] `{認証・認可の確認内容}`
* [ ] データベースを変更した場合、データの保存と取得を確認する

  * [ ] `{登録・更新・削除の確認内容}`
  * [ ] `{マイグレーションの確認内容}`
  * [ ] `{既存データとの互換性の確認内容}`

### 設定・依存関係の確認

* [ ] 実行に必要な設定ファイルと環境変数を確認する

  * [ ] `{必要な設定項目}`
  * [ ] `{必要な環境変数}`
* [ ] 外部サービスまたは外部ファイルを使用する場合、参照方法を確認する

  * [ ] `{外部サービス・ファイル・実行プログラムの確認内容}`
* [ ] 機密情報がリポジトリへ含まれていない

  * [ ] APIキー、パスワード、秘密鍵などがコミットされていない
  * [ ] 必要な除外設定が`.gitignore`などに記載されている
* [ ] 依存パッケージを変更した場合、依存関係が正しく解決される

  * [ ] `{依存関係の復元またはインストールコマンド}`が成功する
  * [ ] 不要な依存パッケージが追加されていない

### Pull Request前の最終確認

* [ ] 今回選択したPull Request単位のタスクがすべて`[x]`になっている
* [ ] 今回の変更目的と無関係な変更が含まれていない
* [ ] 差分を確認し、不要なファイルやデバッグコードが含まれていない
* [ ] Commitが作成されている
* [ ] 作業BranchがリモートリポジトリへPushされている
* [ ] CIが成功している
* [ ] Pull Requestの説明に変更内容、確認内容、関連タスクを記載している
